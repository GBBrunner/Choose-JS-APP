Minimum Requirements

Each project must include the following:

  • A fully functional browser-based app.
  • An intuitive, mobile-first responsive UI.
  • Meaningful functionality that provides value to users.
  • Data persistence using Local Storage or Firebase.
  • Integration of at least one external library.
  • A GitHub repository with clear version control history.
  • A README.md with setup and usage instructions.
  • Unit and integration tests for at least one feature each.

App Features (Choose 5+)

  • User authentication (login/register).
  • Persistent data storage (Local Storage or Firebase).
  • Responsive, mobile-first design.
  • Interactive page with multiple components.
  • Display, filter, and sort stored data.
  • CRUD operations (Create, Read, Update, Delete).
  • Data export options (e.g., JSON file).
  • Feature testing and validation.

Project Workflow

Step 1: Planning

  • Work individually or in pairs (max 2 members).
  • Select your project idea and define sprint goals.
  • Outline key features for the first sprint.

Step 2: Setup

  • Choose a UI framework: Bootstrap, Material, Ionic, or custom CSS.
  • Initialize a GitHub repository and use branching for individual tasks.
  • Create a Kanban board (e.g., Trello or GitHub Issues) to track progress.
  • Agree on a code style guide and naming conventions.
  • Set up CI (Continuous Integration) for automated testing/deployment.

Step 3: Development

  • Sprint 1: Build core functionality ‚Äî UI, data persistence, and main features.
  • Sprint 2: Add advanced features ‚Äî filtering, sorting, exports, and testing.
  • Perform peer code reviews for quality assurance.

Step 4: Finalization

  • Complete, test, and document all planned features.
  • Prepare a presentation showcasing:
    • App purpose and core functionality.
    • Screenshots or live demo.
    • Challenges and solutions implemented.

Step 5: Submission

  • Upload the final project and documentation to GitHub.
  • Post your GitHub repo link in the assignment submission portal.
  • Share your private repo with MountainlandATC.
